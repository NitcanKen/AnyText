# AnyText MVP Requirements

## Product Positioning

AnyText is a personal cross-device relay board for one person using multiple owned devices. It is not a notes app, collaboration tool, long-term database, chat app, or file manager.

The first version solves one job:

> Send Markdown text, code, commands, images, and documents from one owned device to another with less friction than Notion, Discord, WhatsApp, or email.

## Target User

Primary user:

- A developer or technical user working between MacBook, iPhone, iPad, Windows, or another computer.
- Often receives code, commands, explanations, screenshots, or files from ChatGPT.
- Wants the content available on another device immediately.
- Does not want to open a heavy app, create a note, join a chat, or manage a file-sharing workflow.

## Confirmed MVP Scope

### Device Model

- Single-person, owned-device use only.
- No team sharing.
- No public sharing links.
- No collaboration, comments, presence, or permissions.

### Pairing Model

- No account login.
- No Apple, Google, or email sign-in.
- First device creates a device circle.
- New devices join by scanning a QR code or entering a pairing code / room key.
- Pairing is not required for every transfer.
- After pairing, the room key is stored in the browser `localStorage`.
- A device remains joined until the user resets that browser.

### Transfer Model

- Manual-only transfer.
- User opens AnyText, pastes Markdown text, drags in files or selects files, then presses `Send`.
- No clipboard monitoring in MVP.
- No system share extension in MVP.
- No background sync outside the browser app.

### Queue Model

- Content appears in a temporary queue.
- Queue keeps items for 1 hour.
- Multiple items can exist at the same time.
- New items do not overwrite old items.
- Any paired device can manually delete a queue item.
- Items expire automatically after 1 hour.

### Message Structure

Each queue item can contain:

- Optional Markdown text.
- Zero or more image attachments.
- Zero or more document/file attachments.

Limits:

- Markdown text: maximum 500KB.
- Attachments per message: maximum 10.
- Single attachment: maximum 25MB.

### Markdown Requirements

Markdown is a first-class content type.

The receiving view must support:

- Formatted Markdown preview.
- Headings, lists, blockquotes, links, inline code, code blocks, and tables.
- Syntax highlighting for code blocks.
- Copy raw Markdown.
- Copy individual code blocks.
- Stronger visual treatment for shell/command-line blocks.

Any Markdown rendering must sanitize HTML and script content.

### Attachment Requirements

Images:

- Show thumbnails in queue/detail view.
- Support larger preview.
- Support download.

Other files:

- Show file name, size, and type.
- Support download.
- No MVP preview for PDF, Word, Excel, PowerPoint, zip, or other documents.

## Explicit Non-Goals

MVP does not include:

- User accounts.
- Team workspaces.
- Permanent storage.
- Full-text search.
- Tags, folders, categories, or collections.
- Rich text / Notion-style block editing.
- Reordering content blocks.
- Device management table.
- Per-device targeted sending.
- End-to-end encryption.
- Clipboard watching.
- Browser extension.
- Native macOS menu bar app.
- Native iOS/iPadOS app.
- Public file links for other people.

## Security Boundary

MVP uses a lightweight shared secret model.

- A short manual room key is generated by the first device: six digits plus one symbol from `!@#$%^&*`.
- The raw room key is stored only in paired browsers and in pairing links / QR codes.
- Database records should use `sha256(roomKey)` as `roomId`, not the raw room key.
- The short room key is a deliberate UX/security tradeoff for low-friction device pairing.
- Supabase stores message text and attachments in readable form.
- This is not end-to-end encrypted.
- Product copy should clearly say AnyText is for temporary transfer and should not be used for passwords, private keys, or long-term sensitive material.

Baseline safety requirements:

- 1-hour automatic expiry.
- Manual delete.
- Random, hard-to-guess storage paths.
- Sanitized Markdown rendering.
- File size and type validation.
- No execution of received commands.

## Platform Decision

MVP is Web App first.

- Frontend hosted as a static site on GitHub Pages.
- Works on Mac, Windows, iPhone, and iPad through the browser.
- Supabase provides backend database, storage, and realtime sync.

## Success Criteria

The MVP is successful if:

- A new device can join once and then receive future items without repeated QR scans.
- A Markdown answer from ChatGPT can be pasted, sent, rendered, and copied on another device in seconds.
- A command block can be copied individually without selecting surrounding text.
- A screenshot or image can be sent and previewed.
- A document can be sent and downloaded.
- Multiple recent items can coexist in the queue for 1 hour.
- The app feels much lighter than opening Notion, Discord, WhatsApp, or email.
